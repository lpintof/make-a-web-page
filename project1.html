Repetition

One of the most important futures of programming is avoiding repetetion. Repetition is the enemy, or challenge to be taken by a programmer. If it can be repeated, it can be broken down to a process that makes it reiterative. Computers are stupid, thet don't get bored and can do a task as many times as it can, so we should take advantage of that making them do the hard work.

CSS is a great tool to avoid repetition, since it will help us replicate the behavior of one part, across the DOM, whenever we want it to.
Avoiding repetition makes it easier for the programmer to make further changes, and to correct errors faster.

The Document Object Model

More commonly know as DOM, is how web browsers use the info contained in HTML and display it for us.

A great tool to developers is Google Chrome's Developer Tools. In which we are able to see how the webpages "are built".

Boxes within boxes

HTML visually is just boxes within boxes. In which we can divide websites. Modulating our content within boxes, helps us to move them fairly easily and rearrange them according to our needs. Start with the boxes, and refine from there.

To copy a website from a pdf or an image, start by dividing it by boxes. Html will be basically the responsible for them, and CSS will take care of the way they will look.

Order Matters

The hierarchy a browser uses to "style" a website is:

  1. Default Style of browser
  2. Style Sheet in separate file (the main focus of our study)
  3. Stylesheet inside HTML (not recommended)
  4. Inline Style in an element (should be avoided)

Specifics Matter

The more specific a CSS rule is, the more straight it will be applied. Compare to country laws vs, school laws. Chewing gum is legal in the US, in California, in Camarillo, but in Camarillo High School you are not allowed to chew gum. The same happens with CSS, the most specific law is the one that will be applied.

Compare to:

  • Country Law
  • State Law
  • Municipality Law
  • School Law

The most SPECIFIC rule takes effect.

Inheritance

This is a key future of CSS. It simplifies, since we don’t have to specify everything contained in the document. The “child” inherits from the “ancestor” its properties, unless we specify its properties. We return to the idea to avoid repetition.

Read this excrept from wikipedia: “In general, descendant elements inherit text-related properties, but box-related properties are not inherited. Properties that can be inherited are color, font, letter-spacing, line-height, list-style, text-align, text-indent, text-transform, visibility, white-space and word-spacing. Properties that cannot be inherited are background, border, display, float and clear, height and width, margin, min- and max-height and -width, outline, overflow, padding, position, text-decoration, vertical-align and z-index.”1.
